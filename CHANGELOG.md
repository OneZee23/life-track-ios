## [0.5.4] — 28 апреля 2026

### Added
- **Эффект огонька** — иконка 🔥 на overflow-карточке теперь пульсирует (scale 1.0 ↔ 1.18, 0.85s ease-in-out, repeats forever). Подписчик буквально просил «эффект огонька»
- **Подсказка «Нажми, чтобы продолжить»** — на confetti-overlay появилась нижняя строка-намёк, чтобы tap-to-dismiss был обнаружим. Кнопки добавлять не пришлось — сохранили минимализм
- **Inline-редактирование заметок прямо в логе** — раньше была отдельная секция «Заметка на сегодня/вчера» внизу деталки, неудобно читать прошлое. Теперь:
  - Каждая строка лога имеет шеврон `chevron.down` справа
  - Тап по строке разворачивает inline-поле редактирования (`text.bubble.fill` иконка остаётся слева для дней с заметкой — для скан-индикации)
  - Можно читать и редактировать заметку любого прошлого дня прямо в списке
  - При открытии деталки auto-expand'ится строка соответствующая `noteDate` (today/yesterday/specific past day из Progress)
  - Только один день expanded одновременно; flush сохранения при collapse / toolbar Done / dismiss
- **Bubble-icon на карточке теперь интерактивный** — тап по `text.bubble.fill` рядом с именем привычки открывает HabitDetailView (как long-press, но обнаружимее). Параметр `onOpenDetail: () -> Void` в `HabitToggleCard`
- **Bubble-icon в Day и Week Progress** — на DayProgressView рядом с именем привычки появляется иконка если у этого дня есть заметка. На WeekProgressView — если хоть один день недели имеет заметку. Видно сразу в обзоре «здесь есть контекст»

### Changed
- **`HabitToggleCard.onOpenDetail` callback** — добавлен новый параметр для открытия деталей по тапу на иконку заметки
- **Удалена отдельная секция `noteSection`** в HabitDetailView — заменена на inline-редактирование в логе. Removed `noteText`/`lastSavedNote`/`noteSaveTask` state, заменены на `expandedLogDate`/`inlineNoteText`/`inlineLastSaved`/`inlineSaveTask`
- **`ProgressRootView.detailNoteDate` state** — передаётся в HabitDetailView в зависимости от уровня (Day → navDay; Week/Year/Analytics → today)

### Added
- **Кнопка минус для счётчика** — на карточке count-привычки, когда счётчик > 0, появляется маленькая кнопка слева от индикатора. Тап уменьшает count на 1. Случайный лишний тап больше не сбрасывает прогресс целиком — можно отмотать на одно деление
- **Перевыполнение плана** — счётчик больше не упирается в target. После достижения цели можно отметить ещё: 10/9, 11/9 — они подсвечиваются фиолетовым с 🔥 рядом с числом
- **Тап по поздравлению — закрыть** — после полного выполнения дня confetti-overlay закрывается одним тапом по экрану, не нужно ждать таймаут
- **Цель в день без напоминаний** — отдельный toggle «Несколько раз в день» в форме привычки. Можно задать count-цель (3 стакана воды, 9 приседаний) даже если не нужны push-уведомления
- **Расширенная статистика для count-привычек в HabitDetailView**:
  - **Идеальный стрик** (🌟) — отдельный счётчик дней подряд, где `value ≥ target`. Показывается рядом с обычным стриком в шапке
  - **4 stat-карточки**: Всего (sum), Среднее, Лучший день (с фиолетом если overflow), Идеальных дней (с подзаголовком «N сверх плана» если есть overflow)
  - **% выполнения** = `sum(value) / sum(target)` с потолком 100
  - **Bar chart с цветами по уровню**: серый = 0, светло-зелёный = частично, зелёный = ровно target, фиолетовый = overflow. Пунктирная линия target. Тап по бару — tooltip с датой и значением
  - **Календарный heatmap** (новая секция) — 7×N сетка с горизонтальным скроллом, цвет ячейки масштабируется по `value/target`. Включён для count и binary привычек, не для numeric (там и так bar chart)
  - **Лог с `N/T`** — для count-дней показывает количество и target, фиолетовый + 🔥 если overflow
- **Заметки к чекинам** — в HabitDetailView появилась секция «Заметка на сегодня/вчера» с TextField. Можно оставить пояснение к любой привычке (например, «полсигареты в субботу» или «горный поход вместо приседаний»). Хранится отдельно от extended field в новом поле `CheckinExtra.noteValue`. На карточке Check-in появляется маленькая иконка `text.bubble.fill`, если заметка есть на выбранный день
- **`HabitDetailView.noteDate` параметр** — по умолчанию `Date()`. CheckInView передаёт yesterday/today в зависимости от текущей вкладки, чтобы редактировать заметку нужного дня

### Changed
- **`toggleCheckin` без cap** — для count-привычек тап больше не делает кольцевой сброс (`% (target+1)`), просто +1. Уменьшение делается явно через кнопку или новую функцию `decrementCheckin`
- **`HabitToggleCard.onDecrement` + `hasNote` callbacks** — добавлены новые параметры для управляемого уменьшения и индикации наличия заметки
- **Daily target секция отделена от Reminders** — раньше target был внутри секции напоминаний. Теперь две независимые секции, но связь сохранена (включение reminder авто-fill'ит target из количества срабатываний в день, если не настроено вручную)
- **`CheckinExtra.noteValue: String?`** — новое опциональное поле в модели для свободных заметок к чекину. Backward compatible через `decodeIfPresent` — миграции не требуется

## [0.5.2] — 18 апреля 2026

### Added
- **Кастомные эмодзи для привычек** — в сетке эмодзи добавлена ячейка «+», открывающая ввод любого эмодзи с системной клавиатуры (одна валидация — один графемный кластер)
- **Цель в день (count-based привычки)** — в секции напоминаний появился степпер «Цель в день» (1…99). По умолчанию автоматически равен количеству срабатываний в день; если пользователь поправил вручную — изменения расписания больше не перетирают цель
- **Прогресс на карточке Check-in** — для привычек с целью > 1 показывается `N/target` и мини progress bar; тап по карточке = +1, после достижения цели — галочка; ещё тап — сброс в 0

### Changed
- **Мягкий стрик** — при частичном выполнении count-привычки (≥1 отметка в день) стрик продолжается. Полный процент (value/target) влияет только на отображение карточки; в аналитике день засчитан как выполненный, если хотя бы одна отметка проставлена
- **Сигнатура `addHabit` / `updateHabit`** — добавлен параметр `targetPerDay: Int?` (опциональный, совместим с прежним API)
- **Модель `Habit`** — новое опциональное поле `targetPerDay: Int?`; старые данные декодируются как `nil` (обычный toggle-режим), без миграции schema

## [0.5.1] — 16 апреля 2026

### Added
- **Напоминания для привычек** — настраиваемые уведомления: диапазон часов (С/До), интервал (1ч/2ч/3ч), дни недели с быстрыми кнопками «Будни»/«Каждый день». Иконка колокольчика в списке привычек. Бюджет 64 нотификации iOS
- **Foreground-нотификации** — уведомления показываются как баннер даже когда приложение открыто
- **Проверка разрешений** — при открытии формы напоминания автоматически проверяется статус уведомлений; если отключены в Настройках — toggle сбрасывается с подсказкой

### Fixed
- **Баг степпера сна** — HealthKit возвращал дробные минуты (например 420.3), теперь округляются. Шаг степпера 30→15 минут для более точной корректировки
- **Принудительный конфиг sleep/steps** — extended field config для сна и шагов теперь всегда форсируется при сохранении, независимо от ручных настроек
- **Миграция v2** — существующие sleep-привычки получают canonical step=15 через schema migration (безопасное чтение из UserDefaults)

### Changed
- Извлечён `ruPlural` helper для русской плюрализации (дедупликация 3 функций)
- Извлечены `ExtendedFieldConfig.sleepDefault` / `.stepsDefault` константы

## [0.5.0] — 24 марта 2026

### Added
- **Автосинк сна из Apple Health** — длительность сна за ночь (окно 18:00→12:00) автоматически записывается в минутах, отображается как «7ч 50м». Мёрж перекрывающихся интервалов, фильтрация по стадиям сна (core/deep/REM)
- **Автосинк шагов из Apple Health** — точное количество шагов за день, обновляется при каждом открытии приложения
- **Автосинк дистанции тренировок** — для вело/бег/ходьба/плавание/хайкинг автоматически записывается дистанция в км из Apple Health
- **Детальная карточка привычки (HabitDetailView)** — long press на check-in или тап в Progress → экран с бар-чартом (220pt), avg/min/max статами, периодами 7д/30д/90д/Год, линией среднего, тапом на столбик для tooltip, и полным логом значений
- **Навигация в детали** — доступ из Check-in (long press), DayProgress, YearAnalytics, MonthAnalytics
- **Пресеты для числовых привычек** — вместо ручной настройки Unit/Min/Max/Step: готовые пресеты Время (мин), Кол-во (шт), Деньги (₽/$) + кастом
- **Дизайн-система** — `DT` enum с едиными токенами (радиусы, шрифты, размеры), `.healthCard()` modifier с тенями и адаптацией dark mode, `HealthProgressBar` pill-shaped компонент

### Changed
- **Визуальный редизайн всех 7 экранов Progress** — карточки с тенями (Apple Health стиль), pill-shaped progress bars, увеличенные шрифты по единой шкале, heatmap ячейки 14→16px, score circle 72→80pt с radial gradient
- **Форма редактирования привычки** — убран toggle Extended check-in, убраны все numeric settings (Min/Max/Slider/Stepper), убрано превью. Заменено на чипы типа + пресеты. ~400 строк кода удалено
- **Сон хранится в минутах** — вместо часов, отображается как «Xч Yм» через `formatSleepMinutes()`
- **Шаги без округления** — точное значение вместо округления до сотен
- **Контрастность heatmap** — обновлены opacity уровни DayStatus (.20/.45/.70/1.0)
- **NSHealthShareUsageDescription** — обновлено описание доступа к Apple Health

### Fixed
- **Streak в HabitDetailView** — считается на сегодня (`asOf: Date()`) вместо вчера
- **Steps sync no-op** — не пишет в UserDefaults если значение не изменилось
- **Миграция сна** — one-time v0.5.0, флаг ставится до мутации данных, безопасна при крэше

### Removed
- 8 неиспользуемых L10n строк (extendedCheckin, extendedMin, extendedMax, extendedSlider, extendedStepper, extendedInputStyle, extendedPreview)
- Preview section из формы редактирования привычки
- Numeric settings panel (Unit/Min/Max/Step/Slider-Stepper)

## [0.3.0] — 2 марта 2026

### Added
- **Расширенный чекин** — опциональное расширение для привычки: числовой ввод (слайдер/счётчик), текстовый комментарий (до 140 символов), или оценка 0-10. Настраивается в форме привычки
- **ExtendedCheckinPanel** — панель ввода появляется под карточкой привычки при чекине. Числовой: слайдер или ±кнопки с единицей измерения. Оценка: 11 кнопок (0-10, два ряда) с зелёной подсветкой. Текст: TextField с жёстким лимитом 140 символов
- **Опциональный максимум** — для числового типа max необязателен (∞). Без max: только степпер или ручной ввод, слайдер недоступен. Внутренний потолок 99999
- **Ручной ввод в степпере** — тап на число → TextField с decimalPad, точность до 0.01
- **Live-превью** — в форме привычки интерактивная демо-панель: слайдер, степпер, рейтинг или текстовое поле в реальном времени
- **Горизонтальная анимация** — переключение типов (Numeric/Comment/Rating) с боковым слайдом вместо вертикального
- **Отображение extras в DayProgressView** — дополнительные данные показываются под каждой привычкой в просмотре дня
- Новые типы: `ExtendedFieldType`, `NumericInputStyle`, `ExtendedFieldConfig`, `CheckinExtra`
- `checkinExtras` — отдельное хранилище для расширенных данных (UserDefaults, `lt_checkin_extras_v1`)

### Changed
- `HabitFormView` — новая секция настройки расширенного чекина (тип, единица, мин/макс, шаг, стиль ввода) с live-превью
- `addHabit/updateHabit` — расширены для передачи `ExtendedFieldConfig`
- Undo/Redo — включает `checkinExtras` в snapshot
- Extras сохраняются при снятии чекина — случайное снятие/возврат не теряет данные
